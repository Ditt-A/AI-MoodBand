# AI MoodBand

AI MoodBand is a local Flask web app for mood check-ins, curhat-style emotional reflection, memory-aware coaching, optional image context, and daily conversation summaries. The app is written for Indonesian users and uses Gemini models for response generation, summarization, and embeddings.

This project is a non-clinical emotional support prototype. It can help users reflect, organize feelings, and decide small next steps, but it is not a substitute for professional mental health care.

## Features

- Mood-based opening flow for `normal`, `sedih`, `marah`, and `senang`.
- Text-based curhat analysis with emotions, stress score, topics, risk flag, and coach reply.
- Optional image upload for extra context.
- Listen-first response style that asks clarifying questions before giving actions.
- Suggested self-care actions when appropriate, such as breathing, journaling, grounding, breaks, and reach-out steps.
- Optimistic chat UI with typing feedback while the model works in the background.
- Separate saved conversation threads with automatic titles and local history.
- A local analytics dashboard for stress trends, emotions, and conversation topics.
- Local memory store for recent turns and daily summaries.
- Daily summary generation with key points, follow-up prompts, and safety flag.
- Modern Flask UI with responsive styling.

## Tech Stack

## Setup

Create and activate a virtual environment:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

Install dependencies:

```powershell
python -m pip install -r requirements.txt
```

Create your local environment file:

```powershell
Copy-Item API.env.example API.env
```

Fill in `API.env` with your own keys:

```env
API_KEY_GENERATOR=your_gemini_api_key
API_KEY_SUMMARIZER=your_gemini_api_key
API_KEY_TODATABASE=your_gemini_api_key
CSE_ID=
```

`CSE_ID` is present in the example file but is not used by the current app flow.

## Run the App

Start the Flask server:

```powershell
python app.py
```

Open the app in your browser:

```text
http://127.0.0.1:8000
```

The app runs in debug mode on port `8000` by default.

## How It Works

1. The user opens the web page and selects a mood.
2. If the user clicks `Mulai dari Emoji` or submits without text, `app.py` calls `generate_opening()` in `model.py`.
3. If the user submits text, `app.py` calls `analyze_raw()` in `model.py`.
4. `model.py` builds a structured payload with mood, timestamp, profile, recent memory, and optional image data.
5. Gemini returns JSON containing analysis, conversation control, coach reply, suggested actions, and gamification data.
6. The response is rendered into HTML cards and shown in the interface.
7. User and coach turns are saved through `memory_store.py`.
8. Daily summaries are generated by `summarizer.py` and stored as memory records.

## Memory and Data

Conversation memory is stored locally in the `data/` folder:

- `data/memory_meta.pkl` stores metadata and message text.
- `data/memory.index` stores the FAISS vector index when embeddings are available.
- `data/conversations.pkl` stores conversation titles and thread timestamps.

The `data/` folder is ignored by git because it contains local runtime data. Delete the folder if you want to reset local memory.

The current app uses a hardcoded user id:

```python
USER_ID = "demo_user"
```

For multi-user use, replace this with a real authentication or session-based user id.

## Upload Rules

Image uploads are optional. The current Flask route accepts:

- `png`
- `jpg`
- `jpeg`
- `gif`
- `webp`

Maximum upload size is `6 MB`.

## Safety Notes

AI MoodBand is designed as a supportive, non-judgmental reflection tool. The system prompt asks the model to avoid diagnosis, avoid medical promises, and use crisis-aware language when risk is high. The current crisis resources are placeholders such as `<hotline_lokal>` and `<kontak_kampus>`, so production use should replace them with real local resources.

## Troubleshooting

If the app says the model is not ready, check that `API.env` exists and the required API keys are filled in.

If `google-genai` cannot be imported, activate the virtual environment and reinstall dependencies:

```powershell
.\.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
```

If memory search stops working after changing embedding models, delete the `data/` folder so FAISS can rebuild the index with the new embedding dimension.
